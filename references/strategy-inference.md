# Strategy-to-Objective Inference

When the user shares strategic direction during intake (growth plans, hiring,
expansion, fundraising), map those to OBJECTIVES with implied future pain points.

## Explicit strategy → objectives

| User says | Implied objective | Implied future pains |
|---|---|---|
| "Expanding to [country]" | "Serve [country] customers" | Multi-currency, tax compliance, localization, local payment methods |
| "Hiring [N] people" | "Scale team to [N+current]" | Onboarding workflow, task delegation, documentation, access management |
| "Revenue target: [X]" | "Scale operations for [X]" | Current manual processes will break at higher volume |
| "Launching new product/service" | "Support [new offering]" | New tools for the new line, possibly separate process |
| "Preparing to raise funding" | "Investor-ready operations" | Metrics dashboards, clean financials, documented processes |
| "Moving to new location" | "Transition operations" | New local tools, address changes, logistics |

## Inferred objectives (from signals, no explicit statement)

| Signal detected | Inferred objective | Confidence |
|---|---|---|
| Revenue growing >20%/mo | "Scale operations for growth" | MEDIUM |
| Owner does everything alone | "Free the owner from routine work" | HIGH |
| Selling physical goods online | "Automate order fulfillment" | HIGH |
| First international customer | "Serve international customers" | HIGH |
| 10+ disconnected tools | "Simplify tool stack" | HIGH |
| No documentation exists | "Build operational knowledge base" | MEDIUM |
| Seasonal business patterns mentioned | "Handle seasonal peaks efficiently" | MEDIUM |

## How to present inferred objectives

For explicit strategies: present the objective as fact.
"You mentioned expanding to Germany. This implies you'll need: [list]."

For inferred objectives: present as a question.
"Based on your growth rate, you may soon need to scale operations. Is this a priority?"

The user can IGNORE any objective. The output is a menu, not a prescription.
